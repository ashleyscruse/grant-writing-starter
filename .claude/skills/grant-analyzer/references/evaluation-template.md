# Evaluation Template

Use this format when producing grant evaluations. Adjust sections as needed based on the specific opportunity, but maintain this general structure.

```markdown
# {Program Name}

**Funder:** {Agency/Foundation}
**Solicitation:** {URL or reference}
**Evaluated:** {YYYY-MM-DD}
**Decision:** {Strong fit. Pursue. | Moderate fit. Worth exploring. | Weak fit. Skip. | Not eligible.}

## Why This Could Work

{2-4 paragraphs explaining the strongest alignment points between the PI's profile and this opportunity. Be specific: name the research, the publications, the institutional strengths, the track record that maps to what the funder wants. This is the case FOR pursuing.}

## Key Alignment

{Bullet list of the clearest matches between PI profile and grant priorities}

## Gaps and Risks

{Honest assessment of weaknesses. What's missing? What would a reviewer flag? What needs to be built before submitting?}

## Deadlines

{Submission dates, LOI dates, any recurring cycles}

## Open Questions

{Things the PI needs to investigate, decisions to make, people to talk to before committing}

## Recommended Next Steps

{Concrete actions if pursuing: read full solicitation, contact program officer, identify collaborators, develop preliminary results, etc.}

## Notes

{Any additional context: program officer contacts, related opportunities, strategic considerations}
```

## Guidelines

- Lead with the positive case (Why This Could Work) but be brutally honest in Gaps and Risks
- Use the PI's actual publication titles, program names, and institutional details, not generic language
- Reference specific sections of the solicitation when explaining alignment or gaps
- If the grant requires expertise the PI doesn't have, name the type of collaborator needed
- Check the collaborators directory for existing team members who could fill gaps
- Always include deadlines and note if they're recurring
- Keep the evaluation concise: aim for a document someone can read in 3-5 minutes
